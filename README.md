## The Problem

Corporate bankruptcy doesn't announce itself. A firm's financial ratios deteriorate over months before the filing happens — and by the time it does, suppliers have been stiffed, credit lines are frozen, and account teams are left explaining why they didn't see it coming.

This project builds a system that sees it coming. Trained on 10,000 companies with known outcomes and 64 financial ratios per firm, the model scores each of 8,000 new firms on their probability of filing for bankruptcy. The output is not a classification — it is a probability score, ranked and tiered, so that a credit analyst or account manager knows exactly which firms to review and in what order.

The hardest part of this problem is not the modelling. It is the base rate. At 4.73% bankruptcy rate, a model that predicts "safe" for every firm would be 95.3% accurate and completely useless. AUC is the only honest metric here — and hitting 0.9231 on the private leaderboard, 3rd of 37 teams, means the model genuinely discriminates between firms heading toward distress and firms that are not.

<!-- METHODOLOGY -->
<table width="100%" cellspacing="0" cellpadding="0" style="border-left:3px solid #EF4444;background:#06090F;margin:8px 0">
<tr><td style="padding:10px 16px">
<span style="font-family:monospace;font-size:10px;letter-spacing:3px;color:#EF4444;text-transform:uppercase">How the Pipeline Works</span>
</td></tr>
</table>

<br/>

**Step 1 — Preprocessing.** Replace infinite values with NaN. Apply `arcsinh` transformation to all 64 financial ratio columns. This is the key preprocessing insight: financial ratios have extreme outliers and zeros. Log transform fails on zeros. arcsinh handles the full real line — compressing extreme values while preserving sign and working on negative numbers. One transformation choice that moved the needle on AUC.

**Step 2 — AutoGluon with `best_quality`.** This preset runs LightGBM, XGBoost, CatBoost, Random Forest, Extra Trees, Neural Network (PyTorch), and FastAI simultaneously. It uses 8-fold bagging and two levels of stacking — base models at L1, a meta-learner at L2 — with dynamic stacking detection to prevent overfitting. Time limit set to 4 hours to allow full exploration.

**Step 3 — Model selection by profit, not accuracy.** The evaluation metric is ROC-AUC throughout. At a 4.73% base rate, accuracy is meaningless — a model predicting "safe" for everything scores 95.3% accuracy and identifies zero bankrupt firms. AUC measures the model's ability to rank bankrupt firms above safe ones across all possible thresholds.

**Step 4 — Output.** Predicted probabilities for all 8,000 test firms, submitted to Kaggle. Private leaderboard AUC: 0.9231.
